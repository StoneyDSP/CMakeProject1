\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Fixture$>$\newline
class Test\+Caller$<$ Fixture $>$}
Generate a test case from a fixture method. 

A test caller provides access to a test case method on a test fixture class. \mbox{\hyperlink{class_test}{Test}} callers are useful when you want to run an individual test or add it to a suite. \mbox{\hyperlink{class_test}{Test}} Callers invoke only one \mbox{\hyperlink{class_test}{Test}} (i.\+e. test method) on one Fixture of a \mbox{\hyperlink{class_test_fixture}{Test\+Fixture}}.

Here is an example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{class }MathTest : \textcolor{keyword}{public} CppUnit::TestFixture \{}
\DoxyCodeLine{        ...}
\DoxyCodeLine{    \textcolor{keyword}{public}:}
\DoxyCodeLine{        \textcolor{keywordtype}{void}         \mbox{\hyperlink{class_test_caller_ae6880afc711d24ae0b8846759064ceea}{setUp}}();}
\DoxyCodeLine{        \textcolor{keywordtype}{void}         \mbox{\hyperlink{class_test_caller_a0e463b88bf0ceacbd8875e0450ed2649}{tearDown}}();}
\DoxyCodeLine{}
\DoxyCodeLine{        \textcolor{keywordtype}{void}         testAdd();}
\DoxyCodeLine{        \textcolor{keywordtype}{void}         testSubtract();}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{CppUnit::Test *MathTest::suite() \{}
\DoxyCodeLine{    CppUnit::TestSuite *suite = \textcolor{keyword}{new} CppUnit::TestSuite;}
\DoxyCodeLine{}
\DoxyCodeLine{    suite-\/>addTest( \textcolor{keyword}{new} CppUnit::TestCaller<MathTest>( \textcolor{stringliteral}{"{}testAdd"{}}, testAdd ) );}
\DoxyCodeLine{    \textcolor{keywordflow}{return} suite;}
\DoxyCodeLine{\}}

\end{DoxyCode}


You can use a \mbox{\hyperlink{class_test_caller}{Test\+Caller}} to bind any test method on a \mbox{\hyperlink{class_test_fixture}{Test\+Fixture}} class, as long as it accepts void and returns void.
