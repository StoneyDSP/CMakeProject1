\doxysubsection{Detailed Description}
An additional \mbox{\hyperlink{class_message}{Message}} for assertions. 

Provides a implicit constructor that takes a single string. This allow this class to be used as the message arguments in macros.

The constructed object is either a \mbox{\hyperlink{class_message}{Message}} with a single detail string if a string was passed to the macro, or a copy of the \mbox{\hyperlink{class_message}{Message}} passed to the macro.

Here is an example of usage\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{void} checkStringEquals( \textcolor{keyword}{const} std::string \&expected,}
\DoxyCodeLine{                       \textcolor{keyword}{const} std::string \&actual,}
\DoxyCodeLine{                        \textcolor{keyword}{const} CppUnit::SourceLine \&sourceLine,}
\DoxyCodeLine{                        \textcolor{keyword}{const} CppUnit::AdditionalMessage \&message );}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{preprocessor}{\#define XTLUT\_ASSERT\_STRING\_EQUAL\_MESSAGE( expected, actual, message )  \(\backslash\)}}
\DoxyCodeLine{\textcolor{preprocessor}{  ::XtlUt::Impl::checkStringEquals( ::Xtl::toString(expected),        \(\backslash\)}}
\DoxyCodeLine{\textcolor{preprocessor}{                                    ::Xtl::toString(actual),          \(\backslash\)}}
\DoxyCodeLine{\textcolor{preprocessor}{                                    CPPUNIT\_SOURCELINE(),             \(\backslash\)}}
\DoxyCodeLine{\textcolor{preprocessor}{                                    message )}}

\end{DoxyCode}


In the previous example, the user can specify a simple string for {\itshape message}, or a complex \mbox{\hyperlink{class_message}{Message}} object.
