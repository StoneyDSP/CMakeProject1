\doxysubsection{Detailed Description}
Hook to customize Xml output. 

\mbox{\hyperlink{class_xml_outputter_hook}{Xml\+Outputter\+Hook}} can be passed to \mbox{\hyperlink{class_xml_outputter}{Xml\+Outputter}} to customize the \mbox{\hyperlink{class_xml_document}{Xml\+Document}}.

Common customizations are\+:
\begin{DoxyItemize}
\item adding some datas to successfull or failed test with \mbox{\hyperlink{class_xml_outputter_hook_a77310985e055cc2c67e91a42c524fbbe}{fail\+Test\+Added()}} and \mbox{\hyperlink{class_xml_outputter_hook_adbcf6ad2cb85d6f1015306fadb7eadcf}{successful\+Test\+Added()}},
\item adding some statistics with \mbox{\hyperlink{class_xml_outputter_hook_a0e602260274d4f005affb4ee84ce4c4a}{statistics\+Added()}},
\item adding other datas with \mbox{\hyperlink{class_xml_outputter_hook_a1d2b5d5d0fc42d273d6430522a70ccbb}{begin\+Document()}} or \mbox{\hyperlink{class_xml_outputter_hook_a300e2a4ef46db4e76428e32f7c7e6a23}{end\+Document()}}.
\end{DoxyItemize}

See examples/\+Clocker\+Plug\+In which makes use of most the hook.

Another simple example of an outputter hook is shown below. It may be ~\newline
 used to add some meta information to your result files. In the example, the author name as well as the project name and test creation date is added to the head of the xml file.

In order to make this information stored within the xml file, the virtual member function \mbox{\hyperlink{class_xml_outputter_hook_a1d2b5d5d0fc42d273d6430522a70ccbb}{begin\+Document()}} is overriden where a new \mbox{\hyperlink{class_xml_element}{Xml\+Element}} object is created.

This element is simply added to the root node of the document which makes the information automatically being stored when the xml file is written.
